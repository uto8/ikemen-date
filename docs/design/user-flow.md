## 7. チャットフロー

```mermaid
flowchart TD
    A[S13 マッチング一覧] --> |マッチング済み相手をタップ| B[S14 チャット]

    B --> |画面を開いた瞬間| Unread[相手からの全未読メッセージを既読に\nBottomNavバッジをその分減算]

    B --> |テキスト入力→送信| C{入力チェック}
    C --> |空 or スペースのみ| D[送信ボタン非活性\n送信されない]
    C --> |501文字以上| E[「500文字以内で入力してください」\n文字数カウンター表示]
    C --> |1〜500文字| F[メッセージ送信]
    F --> |Supabase Realtime で即時反映| B

    B --> |相手がメッセージ送信| G[Realtime で左側に表示\nページリロード不要]
    G --> B

    B --> |Realtime 接続切断| RC[画面上部に「再接続中...」バナー表示\n自動再接続を試みる]
    RC --> |再接続成功| RR[切断中メッセージをサーバーから再取得\nバナーを非表示]
    RR --> B

    B --> |相手が退会| H[「相手が退会しました」バナー表示\n入力欄・送信ボタン非活性]
    H --> I[過去メッセージは閲覧可能\n送信者は「退会済みユーザー」+デフォルトアバター]

    B --> |不正アクセス\nマッチングしていない相手| J[S13 マッチング一覧にリダイレクト]

    style B fill:#e0f7fa
```

## 8. 退会フロー

```mermaid
flowchart TD
    A[S15 マイページ] --> |「退会する」リンクをタップ| B[退会確認ダイアログ\n「退会すると元に戻せません」]

    B --> |「キャンセル」| A
    B --> |「退会する」を選択| C[退会処理中\nローディング]

    C --> D{処理内容}
    D --> |プロフィール削除\nSupabase Storage の画像削除| E[即時削除]
    D --> |likes レコード全削除| E
    D --> |matches の user_id を NULL に| F[保持・参照をNULL化]
    D --> |messages の sender_id を NULL に| F

    E & F --> G[退会完了]
    G --> |S01 トップにリダイレクト| H[S01 トップ]

    subgraph 相手ユーザー側の影響
        I[マッチング一覧から退会者が消える\nmatches.user_id が NULL のため]
        J[チャット画面に\n「相手が退会しました」バナーが表示される]
        K[過去メッセージは「退会済みユーザー」として残る]
    end

    G -.-> I
    G -.-> J
    G -.-> K

    style H fill:#00c1d4,color:#fff
```

    subgraph データ取得エラー

## 10. 通知バッジフロー

```mermaid
flowchart TD
    subgraph いいねバッジ
        LB1([相手が自分にいいねを送信]) --> LB2{現在表示中の画面は?}
        LB2 --> |S12 以外の任意の画面| LB3[Supabase Realtime で\nlikes テーブル変更を検知]
        LB3 --> LB4[DB に COUNT クエリ再発行]
        LB4 --> LB5[BottomNav いいねタブのバッジを\nリアルタイム更新\n1〜99件: 件数表示 / 100件以上: 99+]
        LB2 --> |S12 表示中| LB6[バッジは表示しない\n画面を開いた時点で全件クリア済み]

        LB5 --> LB7{S12 を開いた?}
        LB7 --> |Yes| LB8[全件一括既読クリア\nバッジを非表示に]
        LB7 --> |No| LB5
    end

    subgraph マッチングバッジ
        MB1([マッチング相手がメッセージを送信]) --> MB2{現在表示中の画面は?}
        MB2 --> |S14 以外の任意の画面| MB3[Supabase Realtime で\nmessages テーブル変更を検知]
        MB3 --> MB4[DB に COUNT クエリ再発行\n自分の送信・退会済み相手を除く]
        MB4 --> MB5[BottomNav マッチングタブのバッジを\nリアルタイム更新]
        MB2 --> |S13 表示中| MB6[バッジを非表示にする\nただしクリアはしない]
        MB2 --> |S14 表示中\n当該マッチングのチャット画面| MB7[バッジを非表示にする\n画面を開いた時点でその分クリア済み]

        MB5 --> MB8{S14 を開いた?}
        MB8 --> |Yes| MB9[そのマッチング分の未読を既読に\nバッジカウントをその件数分減算\n他マッチングに未読があれば継続表示]
        MB8 --> |No| MB5
    end
```

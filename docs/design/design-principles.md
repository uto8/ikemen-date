# デザイン原則 — イケメンデート

## 概要

| 項目 | 内容 |
|---|---|
| ターゲット | 20代前半の女性・スマホ主体・マッチングアプリ経験者 |
| コンセプトワード | 爽やか・清潔感・シンプル |
| 感情目標 | 初めて好きな人と話した日のような、青春の高揚感 |
| 参考サービス | Pairs（カードレイアウト・水色系カラースキームを参考） |

---

## デザイン原則

### 1. 装飾よりも余白を優先する
情報を詰め込まず、余白と行間で「抜け感」を作る。
コンテンツが少なく見えても、余白が清潔感と信頼感を生む。

### 2. ガーリーよりも爽やかを優先する
ピンク・赤・濃い紫は使わない。
水色（`#00c1d4`）を軸に、白・ライトグレーで構成する。
「女性向け」を色で語らず、コピーとレイアウトで語る。

### 3. 機能の主張よりも体験の流れを優先する
ボタンやラベルは最小限。次に何をすべきか、ユーザーが迷わず動ける導線を作る。
タップ対象は最小 44px・指が届く画面下部に配置する。

### 4. トレンドよりも持続性を優先する
グラデーション多用・glassmorphism・アニメーション過多は使わない。
シンプルなフラットデザインを基本とし、半年後も古く見えないUIを目指す。

### 5. 情報量よりも一目で伝わる印象を優先する
カードには「顔・名前・タイプ」の3点のみ。
詳細はタップ後に展開する。最初の画面で決断を迫らない。

## タイポグラフィ

- 日本語フォント: `Noto Sans JP`（weight: 400 / 700）
- 英数字フォント: `Inter`（weight: 400 / 600）
- 本文サイズ: `16px`（`text-base`）
- 行間: `1.7`（`leading-relaxed` 以上）
- 見出しは太字より字間・余白で格を出す（`font-bold` は最小限）

## トーン & ボイス（マイクロコピー方針）

### 基本姿勢
**「友達のお姉さんが優しく背中を押してくれる」** トーン。
丁寧だが堅くなく、明るいが馴れ馴れしくない。

### ルール

| シーン | 良い例 | 悪い例 |
|---|---|---|
| 登録完了 | 「プロフィールができました！さっそく探してみましょう」 | 「登録が完了しました。」 |
| いいね送信 | 「気持ちを伝えました」 | 「いいね！を送信しました」 |
| マッチング成立 | 「つながりました！メッセージを送ってみませんか？」 | 「マッチングが成立しました。」 |
| エラー | 「もう一度試してみてください」 | 「エラーが発生しました。」 |
| 空状態 | 「まだここには誰もいません。もう少し待ってみましょう」 | 「データがありません。」 |

### 禁止ワード
- 「！！」の多用（1文に1個まで）
- 「絶対」「完全無料」などの過剰な断言
- 英語の無理な混じり（「Check it out!」など）
- 古い若者言葉（「ヤバい」「草」など）

## やらないこと

### デザイン
- ピンク・赤・紫をメインカラーに使う
- グラデーション・glassmorphism・neon 系エフェクト
- 画面全体を埋め尽くす情報量
- 横スクロールが必要なレイアウト（スマホで横スクロール禁止）
- 375px 未満の画面を想定したスタイリング

### インタラクション
- タップ後の反応が 100ms を超えるアニメーション
- スクロール中に動くヘッダー（視覚的な揺れを作らない）
- ポップアップの多重表示

### コピー・コンテンツ
- 「出会い系」を連想させるワード（「今すぐ出会う」「近くにいる人」など）
- ユーザーを急かす文言（「今だけ！」「残り〇人！」）
- 性的な表現・過度な身体的描写

## キャラクター画像の使用方針

- オンボーディングの「好みのタイプを選ぼう」画面でグリッド表示
- 各画像にタイプ名バッジを重ねて表示（例: 犬系彼氏 / 塩顔クール）
- 実際のユーザーアバターとは明確に区別するため、登録後の画面には表示しない
- 画像は `public/` 直下のものをそのまま使用（リサイズ・トリミングは CSS で対応）

## S01 — トップ（ランディング）

**目的:** 未ログインユーザーにサービスの価値を伝え、登録またはログインへ誘導する。

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | ロゴ・ログインリンク |
| ヒーロー | キャッチコピー・サブコピー・「無料で始める」CTA ボタン |
| 特徴紹介 | 3点（ギャラリー閲覧・いいね・チャット）をアイコン+説明で |
| フッター | ロゴ・著作権表示 |

### 配置する情報・操作

- ロゴ（左上）
- ログインリンク（右上）
- キャッチコピー（大テキスト）
- サブコピー（補足説明）
- 「無料で始める」ボタン → S02 新規登録へ
- サービス特徴 3点（アイコン + 見出し + 短い説明）
- フッター著作権表記

### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| 正常 | 上記の通り |
| ログイン済みでアクセス | → S10 ユーザー一覧にリダイレクト |

### レスポンシブ方針

- モバイル: ヒーロー全幅・特徴は縦1列
- `md` 以上: 特徴を3カラム横並び

## S02 — 新規登録

**目的:** メール・パスワード・性別・生年月日を入力し、アカウントを作成する。

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | ロゴのみ（ナビなし） |
| フォーム | 入力フィールド群 + 送信ボタン |
| フッター | ログイン画面へのリンク |

### 配置する情報・操作

- メールアドレス入力（type="email"）
- パスワード入力（type="password"・表示/非表示トグル）
- 性別選択（ラジオボタン: 男性 / 女性）
- 生年月日入力（YYYY-MM-DD・ネイティブ date ピッカー）
- 「登録する」ボタン（プライマリ）
- 「ログインはこちら」テキストリンク

### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| 初期 | 全フィールド空・ボタン活性 |
| バリデーションエラー | 各フィールド下にエラーメッセージ表示 |
| 送信中 | ボタンをローディング状態に・重複送信防止 |
| 成功 | → S03 メール確認待ちに遷移 |
| 登録済みメール | 「このメールアドレスはすでに使用されています」 |
| 18歳未満 | 「18歳未満の方はご利用いただけません」 |

### レスポンシブ方針

- モバイル: フォーム全幅
- `md` 以上: フォームを中央寄せ・最大幅 480px

## S03 — メール確認待ち

**目的:** 確認メールの送信を通知し、再送信の手段を提供する。

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | ロゴのみ |
| メインコンテンツ | 送信完了メッセージ + 再送信ボタン |

### 配置する情報・操作

- 送信完了アイコン（メールアイコン）
- 「確認メールを送りました」見出し
- 送信先メールアドレスの表示
- 操作案内テキスト（「リンクをクリックして確認を完了してください」）
- 「確認メールを再送信する」ボタン（セカンダリ）
- 再送信後のクールダウン表示（残り N 秒）
- 「ログイン画面へ戻る」テキストリンク

### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| 初期 | 再送信ボタン活性 |
| クールダウン中（60秒） | ボタン非活性・「あと N 秒後に再送信できます」 |
| 再送信成功 | 「再送信しました」トースト表示 |

### レスポンシブ方針

- モバイル: コンテンツ中央寄せ・全幅
- `md` 以上: 最大幅 480px 中央寄せ

## S04 — ログイン

**目的:** 既存ユーザーがメール・パスワードでログインする。

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | ロゴのみ |
| フォーム | 入力フィールド + ログインボタン |
| フッター | 新規登録・パスワードリセットへのリンク |

### 配置する情報・操作

- メールアドレス入力
- パスワード入力（表示/非表示トグル）
- 「ログイン」ボタン（プライマリ）
- 「パスワードをお忘れの方はこちら」テキストリンク → S05
- 「新規登録はこちら」テキストリンク → S02

### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| 初期 | 全フィールド空 |
| 認証エラー | 「メールアドレスまたはパスワードが正しくありません」 |
| メール未確認 | 「メールアドレスの確認が完了していません。確認メール送信画面へ」→ S03 |
| 送信中 | ボタンローディング |
| 成功（プロフィール登録済み） | → S10 ユーザー一覧 |
| 成功（プロフィール未登録） | → S08 / S09 オンボーディング |

### レスポンシブ方針

- S02 と同様

## S07 — パスワードリセット（新パスワード入力）

**目的:** リセットリンクから到達し、新しいパスワードを設定する。

### 配置する情報・操作

- 「新しいパスワードを設定する」見出し

- パスワード確認入力

### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| 正常 | 上記の通り |

## S08 — プロフィール登録（女性用オンボーディング）

**目的:** メール確認後、女性ユーザーが画像・ニックネーム・居住地を1項目ずつ3ステップで登録して一覧画面に進む。

> モックアップ: `S08-onboarding-female.html`（Step1）/ `S08-onboarding-female-step2.html`（Step2）/ `S08-onboarding-female-step3.html`（Step3）

### 共通レイアウト（全ステップ）

| 領域 | 内容 |
|---|---|
| ヘッダー | ロゴ（Step1はロゴのみ、Step2・3は左端に戻るボタン） |
| プログレスバー | 「ステップ N / 3」テキスト + 3分割バー（完了ステップを primary-500 で塗る） |
| メインコンテンツ | 各ステップの入力UI |
| フッター | 次へ / 登録するボタン（プライマリ・`rounded-full`） |

---

### S08-Step1 — プロフィール画像

**入力項目:** プロフィール画像（任意）

#### 配置する情報・操作

- 「プロフィール画像を設定しましょう」見出し
- 「スキップも可能です」サブテキスト
- 円形アップロードボタン（タップで画像選択）
  - 未選択時: カメラアイコン + 「写真を選ぶ」テキスト・点線ボーダー
  - 選択後: 選択画像を円形プレビュー・右下に編集ボタン
- JPEG / PNG / WEBP・10MB以下 の注記
- 「次へ」ボタン（プライマリ）
- 「スキップする」テキストリンク（Step2へ遷移）

#### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| 初期 | 未選択・アップロードボタン表示 |
| 画像選択後 | 円形プレビュー・右下に編集ボタン |
| アップロード中 | ローディングスピナー |

---

### S08-Step2 — ニックネーム

**入力項目:** ニックネーム（必須・1〜20文字）

#### 配置する情報・操作

- 「ニックネームを教えてください」見出し
- 「本名でなくて大丈夫です」サブテキスト
- テキスト入力フィールド（autofocus・placeholder: 例: はな、あおい）
- 文字数カウンター（現在文字数 / 20文字）
- 「次へ」ボタン（プライマリ）

#### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| 初期 | 空・autofocus |
| 入力中 | 文字数カウンター更新 |
| バリデーションエラー | 「必須項目です」をフィールド下に表示 |

---

### S08-Step3 — 居住地

**入力項目:** 居住地（必須・47都道府県から1つ選択）

#### 配置する情報・操作

- 「住んでいる都道府県を選んでください」見出し
- 「一覧で相手に表示される情報です」サブテキスト
- 都道府県グリッド（3カラム・タップで選択・選択済みは primary-500 ハイライト）
- 「登録する」ボタン（プライマリ）→ 成功時に S10 ユーザー一覧へ

#### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| 初期 | 未選択 |
| 選択後 | 該当ボタンを primary-500 背景・白文字でハイライト |
| バリデーションエラー | 「必須項目です」を表示 |
| 送信中 | ボタンローディング・画像アップロード（Step1 でファイル選択済みの場合）も同時実行 |
| 成功 | → S10 ユーザー一覧 |

---

### レスポンシブ方針（全ステップ共通）

- モバイル: 最大幅 480px・全幅フォーム
- `md` 以上: 最大幅 480px 中央寄せ
- 都道府県グリッド: モバイル 3カラム固定

## S09 — プロフィール登録（男性用オンボーディング）

**目的:** メール確認後、男性ユーザーが7ステップに分けてプロフィール全項目とイケメンタイプを登録する。

| 領域 | 内容 |
|---|---|
| ヘッダー | ロゴ + 戻るアイコン（Step1のみ戻るなし） |
| プログレスバー | 7分割・現在ステップ数をプライマリ色で塗りつぶし |
| フッター | 「次へ」ボタン（Step7のみ「登録する」） |

### Step1 — プロフィール画像（`S09-onboarding-male.html`）

**目的:** 女性が最初に見る写真を設定する。男性は必須。

| 情報・操作 | 詳細 |
|---|---|
| 画像アップロードボタン | タップでギャラリー起動・丸型プレビュー付き |
| 対応形式ヒント | JPEG / PNG / WEBP・10MB以下 |

| 状態 | 内容 |
|---|---|
| 初期 | 破線丸アイコン「写真を選ぶ」 |
| 選択後 | 丸型プレビュー + 編集ボタン |
| バリデーションエラー | 「プロフィール画像は必須です」（スキップ不可） |

> ⚠️ 女性（S08）と異なりスキップリンクなし

### Step2 — ニックネーム（`S09-onboarding-male-step2.html`）

**目的:** アプリ内表示名を設定する。

| テキスト入力 | 最大20文字・文字数カウンター・autofocus |

| 状態 | 内容 |
|---|---|
| 通常 | プレースホルダー「例: 翔太、りく、けん...」 |
| 入力中 | 文字数リアルタイム表示（`N / 20文字`） |

### Step3 — 居住地（`S09-onboarding-male-step3.html`）

**目的:** 相手に表示する都道府県を選択する。

| 情報・操作 | 詳細 |
|---|---|

| 状態 | 内容 |
|---|---|
| 未選択 | 全ボタンがグレー枠 |
| 選択中 | 選択ボタンがプライマリ枠 + bg-primary-50 |

### Step4 — 職業（`S09-onboarding-male-step4.html`）

**目的:** プロフィールに表示する職業を入力する。

| 情報・操作 | 詳細 |
|---|---|
| テキスト入力 | 最大30文字・文字数カウンター・autofocus |

| 状態 | 内容 |
|---|---|
| 通常 | プレースホルダー「例: エンジニア、教師、デザイナー...」 |
| 入力中 | 文字数リアルタイム表示（`N / 30文字`） |

### Step5 — 身長（`S09-onboarding-male-step5.html`）

|---|---|

**目的:** 女性がフィルタリングに使うタイプを自己申告する。

| 情報・操作 | 詳細 |
|---|---|
| タイプ選択グリッド | 2カラム・12種・各タイプにキャラクター画像 + タイプ名 |
| 選択済み状態 | プライマリ枠 + bg-primary-50 + チェックマークアイコン |
| 「登録する」ボタン | 送信後 → S10 ユーザー一覧 |

| 状態 | 内容 |
|---|---|

| 1つ以上選択 | 選択済みカードがハイライト |
| バリデーションエラー | 「1つ以上選択してください」 |
| 送信中 | ボタンローディング |
| 成功 | → S10 ユーザー一覧 |

### レスポンシブ方針

- モバイル（デフォルト）: 全幅・タイプグリッドは2カラム
- `md` 以上: 最大幅 448px 中央寄せ・タイプグリッドは3カラム

## S10 — ユーザー一覧（ギャラリー）

**目的:** 異性ユーザーを2カラムのカードグリッドで閲覧し、詳細へ誘導する。

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | アプリロゴ |
| メインコンテンツ | 2カラム カードグリッド |
| ボトムナビ | 4タブ（一覧・いいね・マッチ・マイページ） |

### 配置する情報・操作

**カード1枚の構成（モバイル）:**
- プロフィール画像（縦長 3:4 比率・`object-cover`・`rounded-2xl`）
- ニックネーム
- 年齢・居住地
- イケメンタイプバッジ（男性カードのみ・最も小さいインデックスの1つ）

**一覧全体:**
- 2カラム グリッド（gap: `space-4`）
- 無限スクロール or ページネーション（未決定）
- 各カードをタップ → S11 ユーザー詳細へ

### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| ローディング | スケルトンカード（2カラム分）を表示 |
| 正常 | カードグリッド表示 |
| 空（ユーザー0人） | 「まだユーザーがいません。もう少し待ってみましょう」 |
| エラー | 「読み込みに失敗しました。再読み込み」ボタン |

### レスポンシブ方針

- モバイル（デフォルト）: 2カラム
- `sm`（640px）以上: 3カラム
- `lg`（1024px）以上: 4カラム

## S11 — ユーザー詳細

**目的:** 特定ユーザーのプロフィール全項目を表示し、いいねを送る。

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | 戻るボタン（← 一覧へ） |
| メインコンテンツ | プロフィール画像 + 基本情報 + いいねボタン |
| ボトムナビ | 共通ボトムナビ |

### 配置する情報・操作

## S14 — チャット

**目的:** マッチング済みの相手とリアルタイムでテキストメッセージを送受信する。

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | 戻るボタン + 相手のアバター + ニックネーム |
| メインコンテンツ | メッセージ一覧（スクロール可能） |
| フッター | テキスト入力欄 + 送信ボタン（固定） |

### 配置する情報・操作

**メッセージ表示:**
- 自分のメッセージ: 右寄せ・プライマリカラー背景
- 相手のメッセージ: 左寄せ・グレー背景
- 送信日時: 当日は「HH:MM」形式・前日以前は「M月D日 HH:MM」形式
- 日付をまたいだ場合は日付区切り線を表示
- 退会済みユーザーのメッセージ: アバターをデフォルトアバター・ニックネームを「退会済みユーザー」として表示

**入力欄:**
- テキストエリア（最大500文字・自動縦伸び）
- 文字数カウンター（入力中に現在文字数/500 を表示）
- 「送信」ボタン（空のとき非活性）

### 状態のバリエーション

| 状態 | 内容 |
|---|---|
| ローディング | スケルトンメッセージ |
| 正常 | メッセージ一覧・最下部にスクロール |
| 空（メッセージなし） | 「メッセージを送ってみましょう」 |
| 送信中 | 送信ボタンローディング |
| 501文字以上 | 入力欄下に「500文字以内で入力してください（現在N文字）」 |
| Realtime切断中 | 画面上部に「再接続中...」バナーを表示・自動再接続を試みる。再接続後はバナー非表示 + 切断中の受信メッセージをサーバーから再取得 |
| 相手が退会済み | 「相手が退会しました。このチャットは利用できません」バナー + 入力欄・送信ボタン非活性 |
| 不正アクセス（マッチングなし） | → S13 マッチング一覧にリダイレクト |
| エラー | 「メッセージの取得に失敗しました」 |

### レスポンシブ方針

- モバイル: 全幅・入力欄はキーボード上に固定（`sticky bottom-0`）
- `md` 以上: 最大幅 720px 中央寄せ

## ボトムナビゲーション（共通）

全ログイン済み画面の下部に固定表示。

| タブ | アイコン | 遷移先 |
|---|---|---|
| 一覧 | Users / GridIcon | S10 ユーザー一覧 |
| いいね | Heart | S12 もらったいいね一覧 |
| マッチング | MessageCircle / Sparkles | S13 マッチング一覧 |
| マイページ | User / CircleUser | S15 マイページ |

- アクティブタブはプライマリカラー（`#00c1d4`）でハイライト
- 非アクティブはサブテキストカラー（`#7a8fa6`）
- 高さ: 64px 固定・セーフエリア（iPhone ホームインジケーター分）を考慮

### 通知バッジ仕様

| タブ | カウント対象 | クリアタイミング |
|---|---|---|
| いいね | 未読のいいね受信数（`likes.is_read = false` かつ自分が受信者） | S12（もらったいいね一覧）を開いた瞬間に全件一括クリア |
| マッチング | 全マッチングの未読メッセージ数（自分の送信・退会済み相手のメッセージを除く） | 各 S14（チャット画面）を開いた瞬間にそのマッチング分のみクリア |

- バッジは 1〜99件はそのままの件数、100件以上は「99+」と表示
- 0件のときはバッジを非表示
- S12 表示中はいいねタブのバッジを非表示にする（開いたとき即クリア）
- S13 表示中はマッチングタブのバッジを非表示にするが、クリアはしない（クリアは各 S14 を開いた時のみ）
- バッジの未読カウントは Supabase Realtime（`likes` / `messages` テーブルへの変更通知）で受け取った際に再クエリしてリアルタイム更新

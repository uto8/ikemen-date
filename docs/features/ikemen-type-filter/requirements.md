# 機能要件書 — イケメンタイプフィルター

最終更新: 2026-05-24

---

## 1. 機能概要

女性ユーザーがユーザー一覧（`/users`）でイケメンタイプを1つ選択し、
そのタイプを持つ男性のみを絞り込んで表示する機能。

- フィルターはユーザー一覧ページ上部に横スクロールの画像カード列として配置する
- デフォルト状態はフィルターなし（全員表示）
- フィルターは女性ユーザーにのみ表示する（男性ユーザーには表示しない）
- フィルター状態はページ内の React state で管理し、URL には保持しない
- 絞り込み後も無限スクロールによるページング（カーソル方式）を維持する

---

## 2. ユーザーストーリー

```
女性として、イケメンタイプを1つ選んで男性ユーザーを絞り込みたい。
そうすることで、自分の好みのタイプの男性だけを効率よく見つけられる。
```

---

## 3. 機能要件

### 3.1 フィルターバーの表示

**画面上の位置**
- ユーザー一覧ページ（`/users`）のカードグリッドより上に配置する
- 横スクロール可能な1行のカード列として表示する
- 各カードは「タイプ画像（`public/{filename}.png`）+ タイプ名」の縦並び構成とする
- カード列の先頭に「すべて」カードを配置する（画像なし・テキストのみ）

**表示対象ユーザー**
- セッション上の `gender` が `'female'` のユーザーにのみフィルターバーを表示する
- `gender` が `'male'` のユーザーには表示しない

**受け入れ条件**

```
Given: 女性ユーザーとしてログインしている
When:  ユーザー一覧画面（/users）を開いた
Then:  ページ上部に12タイプの画像カードと「すべて」カードからなるフィルターバーが表示される
       初期選択状態は「すべて」であり、全男性ユーザーが表示される

Given: 男性ユーザーとしてログインしている
When:  ユーザー一覧画面（/users）を開いた
Then:  フィルターバーは表示されない
       女性ユーザーの一覧が通常通り表示される
```

---

### 3.2 タイプの選択と絞り込み

**選択操作**
- いずれかのタイプカードをタップすると、そのタイプが選択状態になる
- 同時に選択できるタイプは1つのみ
- 選択中のカードをもう一度タップすると選択が解除され、「すべて」状態に戻る
- 「すべて」カードをタップすると、選択中のタイプを解除して「すべて」状態に戻る

**絞り込みの動作**
- タイプを選択した瞬間に一覧を先頭からリセットして再取得する
- 再取得中は既存の一覧を非表示にするのではなく、新しいデータが得られ次第差し替える
- 絞り込み対象は `profile_ikemen_types.ikemen_type_id` に選択したタイプのIDを持つ男性ユーザーとする

**受け入れ条件**

```
Given: フィルターバーで「爽やかスポーツ系」カードをタップした
When:  タップが完了した
Then:  「爽やかスポーツ系」カードが選択状態（枠線 primary-500・背景 primary-50）になる
       ユーザー一覧がリセットされ、profile_ikemen_types に ikemen_type_id = 9 を持つ
       男性ユーザーのみが表示される（新着順・最大20件）

Given: 「爽やかスポーツ系」が選択中の状態で「爽やかスポーツ系」カードを再度タップした
When:  タップが完了した
Then:  選択状態が解除され「すべて」カードが選択状態になる
       ユーザー一覧がリセットされ、全男性ユーザーが表示される

Given: 「爽やかスポーツ系」が選択中の状態で「かわいい系」カードをタップした
When:  タップが完了した
Then:  「かわいい系」カードが選択状態になり「爽やかスポーツ系」の選択が解除される
       ユーザー一覧がリセットされ、ikemen_type_id = 12 を持つ男性ユーザーのみが表示される
```

---

### 3.3 絞り込み結果が0件の場合

**受け入れ条件**

```
Given: フィルターで選択したタイプを持つ男性ユーザーが存在しない
When:  絞り込みの結果が返ってきた
Then:  「このタイプに該当するユーザーはいません」とテキストを表示する
       フィルターバーは引き続き表示される（他のタイプへの切り替えが可能）
```

---

### 3.4 絞り込み状態での無限スクロール

**受け入れ条件**

```
Given: フィルターで絞り込まれた状態で20件目のユーザーカードが画面に表示された
When:  ユーザーが一覧の末尾までスクロールした
Then:  同じタイプフィルターを維持したまま続きの男性ユーザーを20件取得して一覧末尾に追加する

Given: フィルターで絞り込まれた一覧を末尾まで表示している（それ以上の取得対象がない）
When:  ユーザーが一覧の末尾に達した
Then:  追加の取得は行わない（IntersectionObserver の発火を止める）
```

---

### 3.5 フィルター変更時のカーソルリセット

**受け入れ条件**

```
Given: フィルターAで絞り込み済みの状態でカーソル（cursor）が20件目のcreated_atを指している
When:  フィルターBに切り替えた
Then:  カーソルをリセット（null）して、フィルターBの条件で先頭から再取得する
       フィルターAのカーソルをフィルターBの続き読み込みに使わない
```

---

## 4. 非機能要件

| 項目 | 要件 |
|---|---|
| フィルター切り替えの応答性 | タップ後、リセットと再取得を `useTransition` で非同期実行し、UI のブロッキングを発生させない |
| 画像の表示速度 | フィルターバーのタイプ画像は `public/` の静的ファイルを使用し、Supabase Storage への追加リクエストを発生させない |
| フィルター状態の永続化 | ページリロード時はフィルターなし（全員表示）の初期状態に戻る。URL パラメータへの保存は行わない |
| モバイル操作性 | フィルターバーは横スクロールとし、375px 幅でも全12タイプにアクセスできる |

---

## 5. スコープ外

| 項目 | 理由 |
|---|---|
| 複数タイプの同時選択 | 1タイプのみの絞り込みで要件を満たす |
| 都道府県・年齢などの他条件によるフィルター | 本機能の対象外。将来の拡張として別途検討する |
| フィルター状態のURLへの保存 | ページ内 state 管理で十分。ブラウザバック・共有への対応は不要 |
| 男性ユーザーへのフィルター提供 | 男性が閲覧するのは女性一覧であり、女性はイケメンタイプを持たない |
| フィルター結果のソート順変更 | 既存の新着順（`created_at` 降順）を維持する |

---

## 6. 用語

| 用語 | 定義 |
|---|---|
| フィルターバー | `/users` ページ上部に配置する横スクロールのタイプ選択UI |
| 選択状態 | フィルターバーでいずれかのタイプカードが選択されている状態。`selectedTypeId !== null` に相当する |
| 「すべて」状態 | タイプが選択されていない初期状態。`selectedTypeId === null` に相当し、全男性を表示する |
| タイプカード | イケメンタイプ1種を表すUI要素。タイプ画像とタイプ名を縦に並べたカード |
| カーソルリセット | フィルター変更時に `nextCursor` を `null` に戻し、先頭ページから再取得すること |
